### Entity Details

| Entity                | Jurisdiction  | Purpose                                                   | Governance                        |
| --------------------- | ------------- | --------------------------------------------------------- | --------------------------------- |
| **GTCX Foundation**   | Switzerland   | IP ownership, mission protection, constitutional guardian | Foundation board (independent)    |
| **GTCX Technologies** | Portugal      | Core engineering, protocol development, R&D               | Foundation subsidiary             |
| **GTCX Exchange Ltd** | Rwanda (KIFC) | AGX operations, member governance, cross-border trade     | Member-owned (4 classes)          |
| **Regional Offices**  | Various       | Local operations, government relations, field deployment  | Report to Exchange + Technologies |

---

### Why This Structure?

| Design Choice                   | Rationale                                                                             |
| ------------------------------- | ------------------------------------------------------------------------------------- |
| **Swiss Foundation for IP**     | Neutral jurisdiction, credibility, mission protection, open source governance         |
| **Portugal for Tech**           | EU jurisdiction, D2 visa pathway, tech talent, cost-effective                         |
| **Rwanda for Exchange**         | African ownership narrative, KIFC jurisdiction, neutral within Africa, AfCFTA aligned |
| **Regional offices in-country** | Local relationships, government trust, field operations                               |

## Part 2: Governance Architecture

### Member-Owned Exchange (GTCX Exchange Ltd)

| Class          | Who                                      | Governance Weight | Board Seats |
| -------------- | ---------------------------------------- | ----------------- | ----------- |
| **Producer**   | Mining cooperatives, farmer associations | 25%               | 2-3         |
| **Buyer**      | Refineries, traders, manufacturers       | 25%               | 2-3         |
| **Government** | National regulators, ministries          | 25%               | 2-3         |
| **Financial**  | Banks, DFIs, trade finance               | 25%               | 2-3         |

### Board Composition

| Role                             | Selected By      | Term               |
| -------------------------------- | ---------------- | ------------------ |
| Independent Chair                | All members      | 3 years            |
| Producer Representatives (2-3)   | Producer class   | 2 years, staggered |
| Buyer Representatives (2-3)      | Buyer class      | 2 years, staggered |
| Government Representatives (2-3) | Government class | 2 years, staggered |
| Financial Representatives (2-3)  | Financial class  | 2 years, staggered |
| Founding Executive (observer)    | Founder          | Non-voting         |

**Total: 9-13 board members**

### Constitutional Protections

These cannot be changed without supermajority (75%) + class ratification:

- Open source requirement
- Fee caps
- Community benefit distribution
- Producer class floor (minimum 20%)
- No single class can exceed 30%

### Why "Founding Executive" Not CEO?

| CEO Framing                | Founding Executive Framing   |
| -------------------------- | ---------------------------- |
| Implies one person runs it | Implies convening role       |
| Corporate hierarchy        | Multilateral coordination    |
| Permanent position         | Transitional leadership      |
| American/startup framing   | Institution-building framing |

As multilateral infrastructure, the "CEO" role evolves:

- **Years 1-3:** Founding Executive leads coalition formation
- **Years 3-5:** Professional management transition
- **Year 5+:** Board-appointed Executive Director (like SWIFT, DTCC)

## Part 4: Continental Representation Strategy

### The Commitment

GTCX engineering and operations will be **representative of the continent it serves.**

This is not diversity for optics — it's operational necessity:

- Engineers who understand offline-first constraints
- Product designers who know USSD interfaces
- Operations staff with government relationships
- Leaders trusted by producer communities

### Regional Engineering Hubs

| Hub                 | Location                        | Focus                              | Target Headcount (Year 2) |
| ------------------- | ------------------------------- | ---------------------------------- | ------------------------- |
| **Core Platform**   | Portugal (Lisbon)               | Protocol, architecture, security   | 8-12                      |
| **West Africa**     | Ghana (Accra)                   | Field apps, USSD, agent tools      | 6-8                       |
| **East Africa**     | Rwanda (Kigali)                 | Mobile, integrations, localization | 6-8                       |
| **Southern Africa** | South Africa (Cape Town/Joburg) | Enterprise, government systems     | 4-6                       |

### Engineering Team Composition Target

| Region                | Year 1  | Year 2  | Year 3  |
| --------------------- | ------- | ------- | ------- |
| West Africa           | 3       | 8       | 12      |
| East Africa           | 2       | 6       | 10      |
| Southern Africa       | 1       | 4       | 8       |
| Europe (Portugal)     | 6       | 10      | 12      |
| Other (India, remote) | 3       | 6       | 8       |
| **Total**             | **15**  | **34**  | **50**  |
| **% African**         | **40%** | **53%** | **60%** |

**Target: Majority African engineering team by Year 2.**

### Hiring Principles

1. **Local-first hiring** — Hire in-region for regional work
2. **Diaspora recruitment** — Africans abroad who want to return
3. **University partnerships** — Ashesi, ALU, AIMS, Strathmore, UCT
4. **Technical apprenticeships** — Train local talent
5. **Remote-capable** — Distributed team, not HQ-centric

### Compensation Philosophy

| Principle                     | Implementation                                 |
| ----------------------------- | ---------------------------------------------- |
| **Regional parity**           | Top 25-30% of local market, not race-to-bottom |
| **Equity participation**      | All employees get ownership stake              |
| **Cost-of-living adjustment** | Fair pay for where you live                    |
| **No "expat premium"**        | Same role = same equity, adjusted base         |

## Part 6: Operational Management

### How SGX Operations Work

Each country operates their own SGX. GTCX provides:

| GTCX Provides     | Country Provides     |
| ----------------- | -------------------- |
| Software platform | Operating staff      |
| Training          | Government oversight |
| Technical support | Regulatory framework |
| Standards         | Local relationships  |
| AGX connectivity  | Enforcement          |

### SGX Staffing Model (Per Country)

| Role               | Employed By                    | Reports To          |
| ------------------ | ------------------------------ | ------------------- |
| SGX Director       | Government (Ministry of Mines) | Government          |
| Operations Manager | GTCX (seconded)                | SGX Director + GTCX |
| Technical Lead     | GTCX                           | GTCX CTO            |
| Field Coordinators | Government or GTCX             | Operations Manager  |
| Compliance Officer | Government                     | Government          |

**Key principle:** Government controls the SGX, GTCX provides technical capability.
