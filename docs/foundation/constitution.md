# ecosystem-os — repo charter

## Charter

Ecosystem OS is the operating system for running an AI-native company or ecosystem.

## Repo role

| In scope | Out of scope |
| -------- | ------------ |
| Kernel, Graph, Ledger, Surface, Ethos, Venture, Agency, Bridge, Agile, Fabric, Assurance product doctrine and operating specs | Canon publishing ownership · Trade product implementation · Baseline runtime ownership |

## Fleet constitution

This charter operates under the fleet [GTCX Constitution](../../canon-os/docs/governance/constitution/GTCX-CONSTITUTION.md) (Articles I, III, V).

## Non-negotiables

1. **Evidence over narrative** — ship claims require witnesses.
2. **Operating truth over folder history** — repo boundaries can change; product boundaries must stay clear.
3. **Humans and agents share the system** — every autonomous action needs identity, authority, context, and evidence.
4. **Canon publishes** — Canon OS is the documentation and publishing OS; Ecosystem OS operates.
5. **Graph is not RAG** — Graph owns structured operating memory; RAG retrieves source context; MCP acts.
