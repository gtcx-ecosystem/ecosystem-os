# Backend Technology Comparison Analysis

## Purpose

Provide a structured comparison of backend stacks for GTCX services.

## Evaluation Criteria

- Type safety and developer productivity
- Ecosystem maturity and hiring availability
- Deployment flexibility in sovereign environments
- Long‑term maintainability

## Summary

- **Rails**: fast delivery, strong ecosystem, lower type safety
- **NestJS**: TypeScript parity, modular services, higher setup cost
- **Go**: performance and simplicity, slower iteration for complex domains

## Recommendation

Default to Rails where speed matters; use NestJS where protocol integration and type‑safety are critical.
