# Software Development Macro Architecture

## Purpose

Provide a practical abstraction for organizing work from small tasks to system‑level outcomes.

## Hierarchy

- **Task**: the smallest unit of work
- **Feature**: a set of tasks delivering user value
- **Epic**: a sequence of features with a shared outcome
- **Product**: a coherent user experience built from epics
- **System**: multiple products coordinated by shared protocols

## Usage

- Use this hierarchy for roadmap planning and delivery tracking.
- Keep epics tied to measurable outcomes.
- Avoid task sprawl without product‑level alignment.
