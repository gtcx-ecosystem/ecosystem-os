# Research Agenda

## Core questions

1. What is the minimum viable Company Graph for an AI-native company?
2. Which entities and edges should Kernel standardize first?
3. What evidence envelope makes agentic work replayable enough for enterprise trust?
4. Where should RAG stop and Graph begin?
5. Which MCP tools should be first-class product surfaces?
6. How should Ethos represent real human profiles, board, advisors, C-suite, judgment, and ethics without becoming generic HR software?
7. How should Venture track IP, innovation, market theses, and strategic bets?
8. How should Agency connect creative, comms, marketing, demos, and campaigns to Graph and Ledger?
9. What should Surface expose first: CLI, dashboard, graph explorer, work queue, evidence timeline, or agent console?
10. Which module can become the first open-source adoption wedge?

## Validation workstreams

| Workstream | Output |
| ---------- | ------ |
| Company Graph ontology | Entity and edge schema for people, agents, repos, products, work, services, evidence, risks, approvals, revenue. |
| Ledger witness envelope | JSON schema and replay protocol for human-agent work. |
| MCP tool map | First 20 tools across Graph, Bridge, Agile, Fabric, Assurance, Venture, Agency, Ethos. |
| Surface prototype | CLI and console wireframe for next work, graph query, evidence timeline, and service readiness. |
| Competitive matrix | Comparison against Glean, ServiceNow, Salesforce, Workday, Microsoft, Atlassian, Workato, Google, Uniphore. |
| OSS wedge | Decide first open-source package: Kernel schemas, Ledger witness envelope, CLI, or MCP servers. |

## Evaluation lens

Apply the multi-pillar model:

| Pillar group | Research question |
| ------------ | ----------------- |
| Foundation | Can the architecture be governed, implemented, crafted, benchmarked, and trusted? |
| Transformational | Does it create innovation, commercial value, moat, agent empowerment, integration, and IP magic? |
