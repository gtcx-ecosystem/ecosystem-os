# Open-Source Product Strategy

## Principle

Ecosystem OS can be an integrated product while publishing selected modules as open-source products.

Open source should build trust, distribution, and developer adoption. Commercial value should compound in hosted operations, private integrations, enterprise controls, support, evidence retention, and proprietary operating data.

## Candidate open-source surfaces

| Surface | OSS fit | Notes |
| ------- | ------- | ----- |
| Kernel schemas | High | Identity, authority, event, and module contracts can become standards. |
| Graph ontology | Medium | Publish base ontology; keep deployment-specific graph data private. |
| Ledger witness envelope | High | Evidence schemas and replay conventions benefit from transparency. |
| CLI | High | Developer adoption path. |
| MCP servers | High | Natural integration surface for agents. |
| Bridge workflows | Medium | Publish templates and adapters, keep enterprise orchestration hosted. |
| Agile templates | High | Sprint, ceremony, and shipping templates are good OSS adoption hooks. |
| Fabric checks | Medium | Publish selected checks; enterprise service operations stay commercial. |
| Assurance controls | High | Control mappings and evidence schemas benefit from public review. |
| Ethos templates | Medium | Charters, codes, role models can be open templates. |
| Venture and Agency templates | Medium | Publish selected briefs, campaign templates, and IP capture formats. |

## Commercial surfaces

1. Hosted Graph and Surface.
2. Private company graph ingestion.
3. Enterprise Ledger retention and audit exports.
4. SSO, RBAC, approval workflows, policy packs.
5. Managed MCP integration gateway.
6. Service operations packs for Fabric.
7. Assurance packs for compliance and regulated workflows.
8. Implementation, migration, and operating support.

## Adoption path

```text
Open schemas -> CLI -> MCP servers -> templates -> hosted Surface -> enterprise controls
```

The goal is to let a team start with local, inspectable operating primitives and graduate into a managed company operating system.
